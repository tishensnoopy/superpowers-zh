# 子项目 5A：多语言 i18n 设计

- **项目**：佑森小课堂（Yousen Education）官方网站
- **子项目**：5A（多语言 i18n）
- **日期**：2026-07-14
- **状态**：已批准（待规格自检 + 用户审查）
- **前置条件**：子项目 1-4 已完成；Strapi i18n plugin 已开启 zh-CN + en-US

## 1. 背景与目标

### 1.1 背景

佑森小课堂是武汉地区幼小衔接教育机构，拥有 6 个校区。网站已完成中文版本（子项目 1-4），需要支持英文版本以服务外籍家庭、提升 SEO/GEO 覆盖度，并为后续子项目 5B（SEO/GEO）、5C（微信集成）、6A（部署）、6B（多客户）奠定基础。

### 1.2 目标

- 全站支持中英双语（zh-CN 默认 / en-US）
- URL 结构清晰：中文无前缀，英文 `/en-US/` 前缀
- 用户可手动切换语言，记忆偏好（Cookie）
- CMS 内容通过 Strapi i18n plugin 管理双语
- UI 静态文案通过 `next-intl` + `messages/*.json` 管理
- AI 客服跟随页面语言
- 翻译流程：AI 辅助生成草稿 + 人工审核
- 完整 hreflang 标签，避免重复内容惩罚

### 1.3 非目标（YAGNI）

- 不支持中文、英文以外的第三种语言（架构可扩展但不实现）
- 不实现自动检测 Accept-Language 跳转（默认中文 + 手动切换）
- 不实现 Strapi admin 自定义按钮注入（管理员通过外部 curl 调用翻译 API）
- 不实现 AI 客服向量召回质量优化（属 5B 范围）

引入 `[locale]` 动态段，将现有 `app/` 下所有页面（除 `api/`、`robots.ts`、`sitemap.ts`、`llms.txt/` 之外）迁入 `app/[locale]/`：

```
app/
├── [locale]/
│   ├── layout.tsx          ← 由现 app/layout.tsx 迁入，接收 params.locale
│   ├── page.tsx            ← 首页
│   ├── courses/{page.tsx, [slug]/page.tsx}
│   ├── news/{page.tsx, [slug]/page.tsx}
│   ├── campuses/{page.tsx, [slug]/page.tsx}
│   ├── teachers/{page.tsx, [slug]/page.tsx}
│   ├── appointment/page.tsx
│   ├── appointment-success/page.tsx
│   ├── contact/page.tsx
│   ├── faq/page.tsx
│   ├── privacy-policy/, refund-policy/, user-agreement/
│   ├── [slug]/page.tsx     ← 静态单页（关于我们等）
│   ├── error.tsx, global-error.tsx, loading.tsx, not-found.tsx
│   └── globals.css
├── api/chat/{start,message,transfer,history/[sessionId]}/route.ts  ← 保持原位（API 不做 i18n）
├── llms.txt/route.ts       ← 保持原位，内部按 locale 生成双语内容
├── robots.ts               ← 保持原位
├── sitemap.ts              ← 保持原位，输出 hreflang
└── middleware.ts           ← 新增：next-intl 中间件
```

### 2.2 next-intl 核心配置

- **路由策略**：`localePrefix: 'as-needed'` + `defaultLocale: 'zh-CN'` + `locales: ['zh-CN', 'en-US']`
- **URL 表现**：
  - 中文页（默认）：`/courses`、`/news/abc`（无前缀）
  - 英文页：`/en-US/courses`、`/en-US/news/abc`
- **middleware.ts**：处理 locale 检测、URL 重写、静态资源放行
- **根 `app/layout.tsx`**：保留为最小壳层（仅 `<html><body>{children}</body></html>`），实际布局迁入 `app/[locale]/layout.tsx`

### 2.3 Strapi i18n 侧配置

- 已存在 `zh-CN`（default）和 `en-US` 两个 locale
- 启用 i18n 的内容类型共 8 个：
  1. `products`（课程）
  2. `news`（新闻）
  3. `pages`（静态单页）
  4. `faq`（常见问题）
  5. `site-settings`（站点设置）
  6. `navigation`（导航树）
  7. `footer`（页脚）
  8. `testimonials`（家长评价）
- 所有 list/detail API 调用追加 `?locale=<locale>` 参数；缺失 en-US 内容时由前端 fallback 到 zh-CN

## 3. 组件设计

### 3.1 next-intl 基础设施（新增文件）

```
frontend-next/
├── i18n/
│   ├── routing.ts          ← defineRouting（locales、defaultLocale、localePrefix: 'as-needed'）
│   ├── request.ts          ← getRequestConfig（按 locale 加载 messages JSON）
│   └── messages/
│       ├── zh-CN.json      ← UI 文案中文（导航、按钮、表单标签等）
│       └── en-US.json      ← UI 文案英文
├── middleware.ts            ← createMiddleware(routing)，matcher 排除 /api、/_next、/llms.txt、/robots.txt、/sitemap.xml
└── next.config.ts           ← 增加 next-intl/plugin
```

**UI 文案管理策略**：UI 静态文案（按钮文字、表单 label、错误提示等）走 `messages/*.json`；CMS 动态内容走 Strapi i18n API。两者职责清晰分离。

### 3.2 LanguageSwitcher 组件

- **位置**：`components/layout/LanguageSwitcher.tsx`（Client Component，需 `'use client'`）
- **触发位置**：`Navigation` 组件右侧（在「在线咨询」按钮左侧），移动端折叠菜单底部
- **形态**：下拉菜单，展示「中文 / English」
- **行为**：
  - 当前 locale 高亮
  - 切换时调用 `useRouter().replace(pathname, { locale: targetLocale })`，next-intl 自动处理 URL 前缀
  - 写入 Cookie `NEXT_LOCALE=<locale>`（maxAge=1 年），middleware 优先读 Cookie
- **可访问性**：`aria-label="切换语言"`，键盘可操作

### 3.3 Navigation / Footer 改造

- **Navigation**：菜单项 label 由 Strapi navigation 表按 locale 返回；右上区域插入 `<LanguageSwitcher />`
- **Footer**：footer 文案、社交链接 title、版权声明按 locale 渲染
- 两个组件通过 `next-intl/server` 的 `getTranslations()` 在 Server Component 内取翻译，或直接用 Strapi 返回的本地化数据

### 3.4 FloatingChat 双语支持改造

- **props 增加** `locale: 'zh-CN' | 'en-US'`（由父级 Server Component 通过 `useLocale()` 取得后传入）
- **行为变更**：
  - `/api/chat/start` 接受 `locale` 参数，按 locale 选择系统 prompt
  - **系统 prompt 优先级**（与现有 ai-config 设计一致）：`ai-config.systemPromptEn`（en-US）/ `ai-config.systemPrompt`（zh-CN）→ 环境变量 `AI_CHAT_SYSTEM_PROMPT` → 内置默认 prompt
  - 知识库检索 SQL 追加 `AND locale = $<locale>` 过滤（knowledge_base 表新增 `locale` 字段）
  - 转人工提示语、引导留资文案、错误兜底文案走 `messages/*.json`，通过 `useTranslations()`（Client Component）取翻译
- **AI 配置表（ai-config）扩展**：新增 `systemPromptEn` 字段（TEXT），与现有 `systemPrompt`（中文）并列；管理员可在后台分别配置；两者均可为空，为空时 fallback 到环境变量 `AI_CHAT_SYSTEM_PROMPT`

### 3.5 翻译辅助 API（后端新增）

新增 Strapi 自定义 API：`/api/translation/assist`

- **路径**：`backend/src/api/translation/controllers/translation.ts` + `routes/translation.ts`
- **方法**：`POST /api/translation/assist`
- **入参**：`{ sourceLocale: 'zh-CN', targetLocale: 'en-US', contentType: 'products', documentId: string, fields: string[] }`
- **流程**：
  1. 按 documentId 读取 zh-CN 源文档
  2. 调用 DashScope API（复用现有 `qwen-plus` 模型），system prompt 指示"将以下 JSON 字段翻译为英文，保持结构"
  3. 返回翻译草稿 JSON
- **权限**：仅认证管理员可调用（`config/policies::isAdmin`）
- **使用方**：管理员通过 curl/Postman 调用，人工审核后保存为 en-US locale entry

### 3.6 Strapi Admin 翻译按钮（延后）

「AI 翻译」按钮注入到 Strapi Content Manager 编辑页。**默认按延后处理**，避免范围膨胀。管理员可改用 curl/Postman 调 `/api/translation/assist`，人工拷贝结果到 en-US 条目。

## 4. 数据流设计

### 4.1 请求 → locale 检测 → 渲染主流程

```
用户访问 /courses
   │
   ▼
middleware.ts (next-intl createMiddleware)
   │  1. 检查 URL 路径是否带 /en-US 前缀
   │  2. 若无前缀：查 Cookie NEXT_LOCALE → Accept-Language → 默认 zh-CN
   │  3. 若 Cookie=zh-CN 或无 Cookie → locale=zh-CN，URL 保持 /courses
   │  4. 若 Cookie=en-US → 302 重定向到 /en-US/courses
   ▼
app/[locale]/courses/page.tsx (Server Component)
   │  params.locale = 'zh-CN' | 'en-US'
   │  1. setRequestLocale(locale)  ← next-intl 启用静态渲染
   │  2. getProducts({ locale })   ← lib/api.ts 追加 ?locale= 参数
   │  3. getTranslations('CoursesPage')  ← 读取 i18n/messages/<locale>.json
   ▼
React Server Component 渲染
   │  - Strapi 返回的本地化 JSON 直接渲染
   │  - UI 文案通过 useTranslations() / getTranslations() 取
   │  - <html lang={locale}> 由 [locale]/layout.tsx 设置
   ▼
HTML 响应（含 hreflang link 标签，由 buildMetadata 注入）
```

### 4.2 Strapi 内容按 locale 获取

**lib/api.ts 改造规则**（统一模式）：

```typescript
// 改造前
export async function getProducts() {
  const res = await fetch(`${API_URL}/api/products?populate=*`);
  ...
}

// 改造后
export async function getProducts(locale: Locale = 'zh-CN') {
  const res = await fetch(`${API_URL}/api/products?populate=*&locale=${locale}`);
  ...
}
```

**Fallback 策略**（en-US 缺失时）：
- Strapi v5 在 `?locale=en-US` 下若该条目无 en-US 版本，**返回空数组或 404**
- `lib/api.ts` 检测到 en-US 返回空时，**自动重试 `?locale=zh-CN`** 并打 `console.warn`
- 页面顶部显示一条 banner：「此内容暂无英文版，已为你显示中文版本」（文案走 messages JSON）

**影响到的 API 函数清单**（统一加 locale 参数）：
- `getProducts` / `getProductBySlug`
- `getNews` / `getNewsBySlug`
- `getCampuses` / `getCampusBySlug`
- `getTeachers` / `getTeacherBySlug`
- `getPageBySlug`（关于我们等静态页）
- `getFaq`
- `getSiteSettings` / `getNavigationTree` / `getFooter`
- `getTestimonials`

### 4.4 AI 翻译辅助数据流

```
管理员外部 curl 调用 POST /api/translation/assist
   │  body: { sourceLocale:'zh-CN', targetLocale:'en-US',
   │          contentType:'products', documentId:'abc123',
   │          fields:['name','description','content'] }
   ▼
translation controller
   │  1. strapi.documents('api::product.product').findOne({ documentId, locale:'zh-CN' })
   │  2. 提取 fields 字段值，组装 prompt
   │  3. 调用 DashScope /chat/completions（qwen-plus）
   │     system: "You are a professional translator. Translate the JSON values
   │             to English. Keep keys unchanged. Return JSON only."
   │     user: JSON.stringify({ name: "...", description: "..." })
   │  4. 解析响应 JSON，校验字段完整性
   ▼
返回 { translations: { name: "...", description: "...", content: "..." } }
   │
   ▼
管理员审核 → 在 Strapi admin 切换到 en-US locale → 粘贴/编辑 → 保存
   │
   ▼
Strapi lifecycle afterUpdate → 触发知识库同步（按 en-US locale 写入 pgvector）
```

### 4.5 AI 客服双语数据流

```
用户在 /en-US/courses 页面点开 FloatingChat
   │  FloatingChat 组件读取当前 locale（useLocale() from next-intl）
   ▼
POST /api/chat/start  body: { locale: 'en-US' }
   │
   ▼
chat start controller
   │  1. 读取 ai-config 表，按 locale 取 systemPromptEn（en-US）或 systemPrompt（zh-CN）；
   │     字段为空时 fallback 到环境变量 AI_CHAT_SYSTEM_PROMPT，再 fallback 到内置默认 prompt
   │  2. 创建 session，记录 locale=en-US
   ▼
POST /api/chat/message  body: { sessionId, message, locale: 'en-US' }
   │
   ▼
chat message controller
   │  1. 用 session.locale 覆盖入参 locale（防篡改）
   │  2. 向量检索：SELECT ... FROM knowledge_base
   │     WHERE locale = 'en-US' ORDER BY embedding <=> $query LIMIT 5
   │     若 en-US 命中数 < 2 → fallback 查 zh-CN，prompt 追加
   │     "Answer in English. If context is Chinese, translate before answering."
   │  3. 调用 DashScope，temperature/maxTokens 取自 ai-config
   │  4. 返回英文回答
   ▼
FloatingChat 渲染英文回复 + 英文转人工/留资提示
```

### 4.6 知识库 locale 字段同步

**knowledge_base 表新增字段**：
- `locale VARCHAR(10) NOT NULL DEFAULT 'zh-CN'`（已有数据由迁移脚本补齐）

**同步钩子改造**（`backend/src/hooks/knowledge-sync.ts`）：
- Strapi 内容 lifecycle（afterCreate/afterUpdate）触发同步时，按条目 `locale` 字段写入 knowledge_base 表
- 同一 documentId 的 zh-CN 和 en-US 在 knowledge_base 中是**两条独立记录**（不同 id），便于按 locale 过滤检索
- afterDelete 时按 `documentId + locale` 删除对应记录

## 5. 错误处理设计

### 5.1 locale 检测与路由层错误

| 场景 | 处理方式 |
|------|----------|
| URL 中出现不支持的非前缀 locale（如 `/fr-FR/courses`） | middleware 不识别 → 视为普通路径 → 走默认 zh-CN 渲染 → 命中 `not-found.tsx` |
| URL 中出现 `/en-US/` 但实际页面不存在 | `dynamicParams = false` → 404 → `app/[locale]/not-found.tsx`（按当前 locale 渲染） |
| Cookie 中 `NEXT_LOCALE` 被篡改为非法值 | middleware 忽略 → 回退到 Accept-Language → 默认 zh-CN |
| Accept-Language 头格式异常 | try/catch 包裹解析，失败回退到 zh-CN |

**实现要点**：middleware 内所有 locale 来源读取均包 try/catch，任何异常路径都 fallback 到 `defaultLocale`，**绝不抛 500**。

### 5.2 Strapi 内容缺失 fallback

**单条目场景**（如 `getProductBySlug('xxx', 'en-US')`）：
1. 调用 `?locale=en-US` → 若 404 或 data 为 null
2. 自动重试 `?locale=zh-CN`，记录 `console.warn('[i18n] en-US missing, fallback to zh-CN: products/xxx')`
3. 返回结果同时带一个标记字段 `{ data, _i18nFallback: true }`（仅内部使用，不渲染到 DOM）
4. 页面检测到 `_i18nFallback` → 渲染顶部 banner「此内容暂无英文版，已为你显示中文版本」
5. banner 文案走 `messages/en-US.json` 的 `common.i18nFallbackNotice`

**列表场景**（如 `getProducts('en-US')`）：
- Strapi 自动只返回有 en-US 版本的条目 → 列表可能比 zh-CN 短
- **不 fallback**，保持列表纯 en-US
- 列表为空时显示空态文案「暂无英文内容」（走 messages JSON）

**关键约束**：Fallback 只在 detail 页触发，list 页不 fallback，避免出现"列表有但点进去是中文"的混乱体验。

### 5.3 翻译辅助 API 错误处理

`POST /api/translation/assist` 错误码：

| HTTP | 场景 | 响应体 |
|------|------|--------|
| 400 | contentType 不在白名单 / fields 数组为空 / documentId 缺失 | `{ error: { code: 'INVALID_PARAMS', message: '...' } }` |
| 401 | 未认证 | `{ error: { code: 'UNAUTHORIZED' } }` |
| 403 | 已认证但非管理员 | `{ error: { code: 'FORBIDDEN' } }` |
| 404 | documentId 在 zh-CN 下不存在 | `{ error: { code: 'SOURCE_NOT_FOUND' } }` |
| 502 | DashScope API 调用失败/超时（30s） | `{ error: { code: 'AI_PROVIDER_ERROR', message: '上游服务异常' } }` |
| 502 | DashScope 返回内容非合法 JSON | `{ error: { code: 'AI_RESPONSE_PARSE_ERROR' } }` |

**重试策略**：DashScope 调用失败时**不重试**（避免双倍计费），直接返回 502 让管理员手动重试。

**字段完整性校验**：AI 返回的 JSON 必须包含入参 `fields` 的所有 key，缺失则返回 502 + `AI_RESPONSE_INCOMPLETE`，并记录缺失字段到 `console.error`。

### 5.4 AI 客服双语错误处理

| 场景 | 处理方式 |
|------|----------|
| `session.locale` 在 DB 中被篡改为非法值 | 回退到 zh-CN 路径（`ai-config.systemPrompt` → 环境变量 → 默认）+ 检索不按 locale 过滤，记录 `console.warn` |
| en-US 知识库命中 0 条 + zh-CN fallback 也失败 | 返回英文兜底语「Sorry, I cannot answer this question. Would you like to transfer to a human agent?」（走 messages JSON） |
| 知识库 `locale` 字段为 NULL（历史数据迁移期） | 视为 zh-CN（DEFAULT 'zh-CN' 已保证），不影响检索 |
| FloatingChat 收到未知 locale | 默认按 zh-CN 处理，记录 `console.warn` |

### 5.5 i18n messages JSON 缺失 key

**场景**：`en-US.json` 中缺少某个 key（如 `CoursesPage.title`）。

**next-intl 行为**：默认抛出 `MISSING_MESSAGE` 错误 → 页面 500。

**我们的策略**：
- 开发环境：保留 next-intl 默认抛错，便于发现遗漏
- 生产环境：`onError` 回调降级为 `console.error` + 返回 key 本身，**不抛 500**
- 配置方式（`i18n/request.ts`）：

```typescript
getRequestConfig: async (locale) => ({
  messages: (await import(`./messages/${locale}.json`)).default,
  onError: (error) => {
    if (process.env.NODE_ENV === 'production') {
      console.error('[i18n]', error.message);
    } else {
      throw error;
    }
  },
  getMessageFallback: ({ key }) => key,
})
```

### 5.6 知识库迁移期错误

**风险**：knowledge_base 表新增 `locale` 字段时，已有 16 条向量记录的 `locale` 为 NULL（DEFAULT 子句只对新插入行生效）。

**处理**：
- 迁移脚本 `backend/scripts/migrate-knowledge-base-locale.ts`：`UPDATE knowledge_base SET locale = 'zh-CN' WHERE locale IS NULL`
- 该脚本在 5A 实现阶段作为首个迁移任务执行，**先迁移再上线代码**
- 迁移后启动应用，后续所有同步钩子写入时强制带 `locale` 字段

### 5.7 Sentry 错误上报约定

- i18n fallback 事件（en-US → zh-CN）**不上报 Sentry**，仅 `console.warn`（属于业务正常路径）
- `/api/translation/assist` 的 5xx 错误**上报 Sentry**，附带 `contentType`、`documentId`、`code`，**不上报** 翻译原文和 AI 响应体（可能含 PII）
- AI 客服 locale 异常**上报 Sentry**，附带 `sessionId`、`attemptedLocale`
- messages JSON 缺失 key 在生产环境的 `console.error` **不上报 Sentry**（避免噪音），由 E2E 测试保障覆盖

## 7. 实现顺序与依赖

实现阶段将按以下顺序推进（详细任务拆分由 writing-plans 技能生成）：

3. **后端 chat 双语**：start/message locale 透传 + 知识库 locale 过滤 + 单元测试
4. **前端 next-intl 基础设施**：i18n/routing.ts + i18n/request.ts + middleware.ts + messages JSON
5. **前端目录迁移**：`app/` → `app/[locale]/` + 根 layout 壳层化
6. **前端 lib/api.ts 改造**：所有函数加 locale 参数 + fallback 逻辑 + 单元测试
7. **前端组件改造**：LanguageSwitcher + Navigation/Footer + FloatingChat
8. **SEO 改造**：sitemap.ts + seo.ts hreflang + llms.txt 双语
9. **Strapi seed 扩展**：en-US 占位内容
10. **E2E + 视觉回归测试**：i18n.spec.ts + i18n-seo.spec.ts + i18n-chat.spec.ts + visual-i18n.spec.ts

## 8. 风险与边界

### 8.1 已知风险

| 风险 | 缓解措施 |
|------|----------|
| `app/` → `app/[locale]/` 迁移可能影响 SSG/ISR 配置 | 每个页面 `generateStaticParams` 显式列出 locales；`dynamicParams = false` 保留 |
| next-intl 与现有 Sentry 集成冲突 | 早期 E2E 验证 Sentry 仍正常捕获错误 |
| Strapi i18n plugin 在 list API 行为与预期不符 | 实现前先手动 curl 验证 `?locale=en-US` 返回行为 |
| 知识库迁移期间 AI 客服检索异常 | 迁移脚本作为首个任务执行，迁移完成前不部署代码改动 |
| messages JSON key 漏译 | 双向完整性单元测试 + E2E 覆盖 |

### 8.2 与其他子项目的边界

- **5B（SEO/GEO）**：本子项目只实现 hreflang 标签；GEO 优化（llms.txt 内容质量、结构化数据、AI 摘要）属 5B
- **5C（微信集成）**：本子项目不涉及微信；微信内嵌页面默认走中文 locale
- **6A（部署）**：本子项目不修改 docker-compose.yml；Docker 构建流程不变
- **6B（多客户）**：本子项目仍使用 `yousen-*` 容器名；多租户隔离属 6B
